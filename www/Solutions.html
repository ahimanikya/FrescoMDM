Mural Industry Models
June 2008

To help you get quickly started using Mural applications for specific tasks, we created a set of models, or templates, that you can download and use to create a master index application that is already geared to a specific type of data. We identified common models and requirements for different industries and created templates based on those unique needs. Note that this is a work in progress, and we will continue to create additional models and make them available for downloading.

The models demonstrate how certain data types can be implemented in the MDM Suite. The data structure and the standardization and matching rules are based on the specific requirements of the type of data being processed. For example, both the United State and Australia patient models use first and last names for blocking and matching, but the United States version also uses the Social Security Number, while the Australian version uses a Tax File Number.

Currently, the following models are available for you to download and use. You are welcome to share your models here as well. A solutions template is provided for you to use when developing your own solutions.

United States Patient Model

The United States Patient model is based on the specific data processing requirements for a healthcare implementation in the United States. To learn about how the model was designed and configured, see United States of America Patient Solution.

Australia Patient Model

The Australia Patient model is based on the specific data processing requirements for a healthcare implementation in Australia. To learn about how the model was designed and configured, see Australian Patient Solution.

United Kingdom Patient Model

The United Kingdom Patient model is based on the specific data processing requirements for a healthcare implementation in the United Kingdom. To learn about how the model was designed and the configuration that was used, see United Kingdom Patient Solution.

To Download the Solutions

Before you begin, make sure you have NetBeans and a Mural build installed. For instructions, see Installing Mural Builds.

  1. Navigate to the Mural Downloads page.
  2. Under Solutions, click the name of the model you want to download.
  3. Save the file to your disk, and then extract the file into your NetBeans projects home directory.
  4. Launch the NetBeans IDE.
  5. Right click in the Project window, and click Open Project.
  6. Navigate to the location where you extracted the model files, and select the name of the Project.
  7. To start working with the model, view the readme file that is included in the model ZIP file.
