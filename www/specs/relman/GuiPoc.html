Performance Test


I created a quick performance test where I showed 10,022 nodes on a tree.  Most nodes on the tree have about 3 - 4 children. One area has 10,003 children.

All nodes where defined at start up so this does not include any additional time it would take if the tree was implemented using a Lazy Load.

Machine:

VMFusion running Windows XP with Service Pack 3
Intel (R) Xeon(R) CPU
E5462 @ 2.80 GHz
2.78 GHz, 2.00 GB of RAM

Results:

Timed using system clock so these are rough.

Expand a node without a lot of children - 3 seconds

Expand the node with 10,003 children - 10 seconds

Conclusion:

Showing trees with large amounts of data is not in the three seconds you would like. 

Recommendation:


We should run a similiar test using either the DoJo or Yahoo UI tree to compare results.
