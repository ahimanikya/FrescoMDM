Administer->Groups

Define what domains are in a group. Also define what attributes are with each group relationship.

Use Cases 

  • View and maintain group definitions for a specific domain.
  • View and maintain attributes that are associated with a specific group instance.  A group instance is a set of records that share a common attribute, such as a household.  An examples of group instance attributes are  "number of residents" and "original occupent". 
  • View and maintain custom attributes for records that belong to a group instance.  An example of a group record custom attribute is "household head" which would be true or false. 

Screen

  • Similar to the other administer screens except that this will have separate tabs to define attributes for the group instance and attributes for the custom records. 

Security Operations


Operation Effect Must Implement
AdminGroups_ReadWrite Allows user to view, add, delete and updatee relationships and attributes associated with a specific relationship.
Yes
AdminGroups_ViewOnly View only mode of relationship and Add / Edit relationship dialog.   If user has access to this operation and not the AdminRelationships_ReadWrite operation then View... is shown by each relationship instead of Edit.
No

Notes


  • If neither operation is assigned to a user than the Administer->Groups tab is not shown.
