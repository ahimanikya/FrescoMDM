Manage->Relationship->Maintain->By Record  

Use Cases

  • View and maintain relationships for a single record and all relationships to all associated records.  This includes
    • Deleting relationships
    • Adding relationships
    • Moving relationships  from one record to another record in the same domain.  Example,  a Location that was Owned by Centerpoint is now owned by Global Enterprise. 
    • Moving all relationships associated with one record to a different record.  Example:   Example, Dr. Andrea Smith takes over for Dr. George Karlin and all the relationships that use to be associated with Dr. George Karlin are now associated with Dr. Andrea Smith. 
  • View details of selected relationship (source record, destination record and relationship attributes)

Notes on Tree Usage


  • Selecting a relationship type allows deletion of all the records in the relationship.
  • Selecting a domain disables Delete and Add.
  • Allow for multi-selected.  If more than one record is selected then the right hand side should be blank and the number shown by Delete should be updated. 
  • Move could be implemented by finding a record and moving all selected records under the record (similiar to how the hierarchy tree works) or better yet by showing a second tree on the right that just displays a Find Record button (and hide) and allowing for drag / drop between the two trees (current on right, new on left)
  • Do not  allow any record that is already in its parent path to be expanded (stop cyclic references)
    Example:  Below "Dr. George Karlin" is not expandable because the record is already expanded.

    If Centerpoint->Bangalore had children relationships that refered back to Centerpoint than that node would not be expandable.
  • Build in a configurable limits to the number of records to show under a node.  If too many records would be shown then the tree would show a paginated table of the nodes and a More...  button (for when the table is dismissed). For the session if a record is selected from that table it appears in the tree list and under the parent node prior to the More... button.

    Nice to have would be the abliity to filter the table as follows:

    • Names of destination records (optional)
    • Relationship attributes (optional)

Move Dialog


Only available if nodes under the same parent have been selected or a single node has been selected.  

The user is given choices of where to move the node/s based on the type of node selected as follows.

Relationships 
The dialog shows a message indicating what will be moved.  Example, Move relationships associated with the 2 selected relationships to a new Company.

A search screen for the same domain as the parent record of the relationship is also shown.  The user can then select the new record to re-assign all records that share the relationship to the original record. 

Example, the user selects  -> Owns Offices and <- Employed By relationships under Centerpoint.  The move dialog shows a search screen for Company domain.  The user can search for the record to that will replace Centerpoint for these two relationships.

Domains
The dialog shows a message indicating what will be moved.  Example, Move relationships associated with the selected domain to a new Company.
User is presented with a search screen for the same domain as the parent record of the relationship.  The user can then select the new record to re-assign all records that share the relationship to the original record. 
Example, the user selects Locations under Company > Centerpoint > Owns .  The move dialog shows a search screen for the Company domain. from which the user selects teh record to re-assign all the Locations under Company > Centerpoint > Owns to.

Records
The dialog shows a message indicating what will be moved. Example,  Move selected Centerpoint -> Owns Locations to another Company.
User is presented with a search screen for the same domain as the parent record of the relationship.  The user can then select the new record to re-assign all records that share the relationship to the original record. 
Example, the user selects records unde   Company > Centerpoint > Owns > Locations .  The move dialog shows a search screen for the Company domain from which the user selects the record to re-assign the Company > Centerpoint > Owns >  relationships to.

 
In other words the dialogs work similar to the Rules for allowable actions on Relationship Tree. The difference being that instead of using drag / drop the user is guided by the Move dialog to choices that make sense given the context of the currently selected nodes
