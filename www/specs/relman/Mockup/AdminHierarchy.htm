Administer->Hierarchies

Provides a template for hierarchies of records within a domain. The user can view and maintain these templates which are then used by the maintenance hierarchy screens.

Use Cases

  • View and maintain hierarchy templates
  • View and maintain attributes associated with the hierarchy. 
  • View and maintain attributes that are associated with the record belonging to a specific hierarchy - the hierarchy node.

Implementing the Available Progress Bars


The progress bars are considered a "nice to have" feature.    They inform the user on the number for each datatype that is available to use as a custom attribute.  

When the user adds an attribute of a certain type the value in the bar is increased by one.  When the bar gets to 100% than the specified type can not be selected from the type pull down.  

Other updates to the bar are made when an attribute is deleted or the type is changed.  Deleting an attribute decresses the available count by one.  Changing the type decresses the available count for the old type by one and incresses the count for the new type by one.  (Example, changing "AString" to type char would make the strings count 1 of 20 and the char count 76 of 100).
