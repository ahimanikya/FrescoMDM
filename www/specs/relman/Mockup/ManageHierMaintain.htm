Manage->Hierarchies->Maintain

Used to view and maintain an existing Record Hierarchy.

Use Cases

  • Maintain a hierarchy of records within the same domain (Note: the same record may only appear once in a specific heirachy) 
  • View the position of a record within the hierarchy
  • View details about a record within a hierarchy
  • View the properties (fixed and custom attributes) associated with a hierarchy.
  • Move (re-arrange) records

The screen allows a user to both view and maintain the record hierarchy. It also allows a user to view details about the records

3. View Initial record

Displays information about a record selected from the search screen.  

In this case "James Mason" was selected.   

Note: If the root node option was selected the the first node in the hierarchy (if any) is shown.


The specific hierarchy that is displayed is determined by the currently selected access control for the user. Note: In other words a hierarchy is associated with an access control and not a specific user.

Notes

  • An actual hierarchy tree would show all the records at the same level. However the only branches to be expanded would be the one representing the path to the selected node in the hierarchy.
  • If the current access role allows for update of the hierarchy then records may be arranged on the tree. In addition to the buttons a group of records may be selected and moved via drag/drop.

OperationEffectMust Implement
ManageHierarchies_ReadWrite
  • User has access to all buttons as shown in the mock up.  
Yes
ManageHierarchies_ViewOnly
  • The only buttons a user can choose are "Select", "Find", "Filter", "View Summary" and "View Details". 
  • The Add, Move, Delete, Replace and Rearrange buttons are not shown.  
  • The tree only supports a single select to view relationship and record details.  Drag / Drop on the tree is disabled.
  • The relationship attributes are read only so neither the Save nor the Revert button is displayed for these.
