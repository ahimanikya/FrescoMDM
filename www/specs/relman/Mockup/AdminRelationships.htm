Administer->Relationships

Provides a template for linking relationships within or across domains. A relationship is associated with a relationship type like ManagerOf and the source and target domains.

Use Cases

  • View and maintain relationship templates including the 'direction' of the relationship (A -> B, A <--> B, note B -> A is implied)
  • View and maintain attributes that are associated with the relationship, such as how long the relationship is good for.
  • View and maintain extended attributes for the specific relationship.
  • Define

Issues

  • Should the extended attributes be defined in the Netbeans GUI? The perfect solution would be that once a service was deployed once it did not have to be deployed again in order to update attributes. In this case the Netbeans GUI would connect to the running service after it was initially deployed in the same way as the Web Application. If this can not be implemented than Netbeans is the correct place to handle the creation of the extended attributes as these would certainly be used by the Relationship rules editor.

    Answer: Extended attributes shall be definable in both the Netbeans GUI and on the Web. It is important to note that extended attributes once used can not be removed. Additionally extended attributes are defined for each relationship type. Future enhancements might include keeping track of a set of common extended attributes.
