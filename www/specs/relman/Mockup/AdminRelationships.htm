Administer->Relationships

Provides a template for linking relationships within or across domains. A relationship is associated with a relationship type like ManagerOf and the source and target domains.

Use Cases

  • View and maintain relationship templates including the 'direction' of the relationship (A -> B, A <--> B) 
  • View and maintain fixed attributes that are associated with the relationship, such as how long the relationship is good for.
  • View and maintain custom attributes associated with the specific relationship.

Notes

  • If moving several items at a time via the Move Up/Down  function acts as if each line was moved separately.  I.E. For Move Up - Adate would move before AString and  Afloat would move before AnInt.
  • Clone features are 'nice to have'.  These would bring up the entire list for the associated 'browse' function so a user could select and clone a definition.
  • Searchable options:  By Range, By Value and No. (Default No)."

    By Value.   In any search / select screen that a user can enter a custom attribute for a relationship the user can only search for the single attribute value.  Thus the search screen should only show a single entry area for the attribute.

    By Range.  In any search / select screen that a user can enter a custom attribute for a relationship the user can search for a relationship that has that value by specifying a range or a single value.  If a custom attribute is specified as "By Range" than the search screen should a Drop-Down list by the customer attribute with a: 'By Value', 'By Range' selection. (Default: By Range).  If the user selects the By Value option than an entry field for the form is shown.  If the user selects the By Range option than a From: [entry-field]   To: [entry-field] is also shown.

    No. The custom attribute will not appear in any search screen.  It will only appear in a Relationship Attribute section of a screen.

    Once a selection is made the Web GUI should remember the preference by Relationship Type and Custom Attribute for the Session.
