General layout and masthead

Use Cases

  • Provides the ability to change the access role that the user is currently viewing and maintaining relationships with. A user may want to change the access role in order to prevent accidental update of a relationship. Another reason a user may want to select another access role is to see a different view of the same data. For example, a user with an access control of 'Sales' might not be able to view certain relationships. A user with an access control of 'CEO' might be able to view relationships that are consider sensitive.
  • Provides a way to change the valid dates that a relationship can be viewed. The dates act as a filter such that only active relationships are shown (relationships that are valid within the date range).
  • General layout and masthead

Notes:

  • While the preference is to have the select record area to the left there are causes where the select area must be on the top.
  • Administrator screens do not need a separate select area.
  • The main area normally is used for showing record details and relationship details.
