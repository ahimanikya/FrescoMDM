Future Ideas

Plans for our next generation Open Source MDM solution

Relationship Manager


Get more value from the clean data by defining and querying relationships between Single Best Records.  

Relationships include
  • M to N Relationship between record instances in the same domain or in different domains.  Example:  InsuranceCompany Insures USAPatient.
  • Hierarchical organization of record instances in the same domain.  Example:  Organizational Chart of Persons
  • Groups arbitrary groupings of record instances in the same domain.  Example:  People in a Household. 
  • Categories taxonomy of records.   Example:   Books->Genre->East Coast.
For more information click here.

Multi-tier Models


Support arbritrary number of levels in a domain model allowing for more natural organization of fields.  Exampe:  UKPatient->Condition->MedicineTaken
