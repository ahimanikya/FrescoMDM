Manage->Hierarchies->Maintain

Used to view and maintain an existing Record Hierarchy.

Use Cases

  • Maintain a hierarchy of records within the same domain
  • View the position of a record within the hierarchy
  • View details about a record within a hierarchy
  • View the properties (fixed and extended attributes) associated with a hierarchy.

The screen allows a user to both view and maintain the record hierarchy. It also allows a user to view details about the records

1. Initial screen


Initial screen indicates that either a domain and hierarchy must be selected or a user must search for a record.

2. Search for Record

In this example, the Person domain has a field called 'Group'.  Thus the Group below is not a group type but a field of the domain.  

The use case shown here is to locate the manager for the group called 'Mural' and finding out information about the people that work for him.  

First we search the domain person for everyone that belongs to the group Mural.

The screen here only shows three record fields. However it might have been defined to also show additional fields such as a primary role of a person.

Either way after the results of the initial search are shown the user selects the record they want to view more details for.


Notes

  •  In the final version it is unclear if we will show a summary of field value or a record identifer.  For the search I prefer the summary so that a user can have some degree of confidence that they located the correct record.

3. View initial record

Displays information about a record selected from the search screen.  

If Domain and hierarchy are selected then the root node is shown.

The specific hierarchy that is displayed is determined by the currently selected access control for the user. Note: In other words a hierarchy is associated with an access control and not a specific user.

Notes


  • An actual hierarchy tree would show all the records at the same level. However the only branches to be expanded would be the one representing the path to the selected node in the hierarchy.
  • The relationship name showed for the corresponding parent or child relationship is determined by the relationship type definition.
  • If the current access role allows for update of the hierarchy then records may be arranged on the tree. In addition to the buttons a group of records may be selected and moved via drag/drop.

Issues
