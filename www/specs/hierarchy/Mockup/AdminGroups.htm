Administer->Groups

Define what domains are in a group. Also define what attributes are with each group relationship.

Use Cases 

  • View and maintain group definitions.
  • View and maintain attributes that are associated with a specific group -> domain relationship. Such as how long the relationship is good for. i.e. how long is a record in a group.
  • View and maintain extended attributes a specific group -> domain relationship.

Andrea - A group is just a name.  Each domain - group has its own set of attributes.  The screen represents that a users thinks of the group name first, not the domain.  Example, I can have a group called household associated with a UK Patient.  I can have a different group called household associated with USA Patient.  

So while I expect that the group is stored for a domain I believe it would be nice to implement the groups as per this screen.  What this suggest is a convenience API that gives all the groups defined for all the domains.   If we can not then we can have a list of domains first.  And then add all groups for that domain.
