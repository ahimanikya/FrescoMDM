Configuring the Environment

Before you start working with the IBML Tool, make sure the environment is configured to handle the requirements of the load process.

To Configure the Environment

  1. Complete the steps under Generating the Initial Bulk Match and Load Tool.
  2. If you are using a distributed processing environment, set up an FTP server on the master processor using the FTP provider of your choice.
  3. Install the Oracle or SQL Server database driver on each processor running the IBML Tools.
  4. If the master index database is an Oracle database, define an environment variable:
    • For Windows, create an environment variable named ORACLE_JDBC_JAR. Define the value of the variable as the path and name of the driver file, ojdbc14.jar; for example, C:\oracle\jdbc\lib\ojdbc14.jar.
    • For UNIX, run export ORACLE_JDBC_JAR=oracle_driver_path/ojdbc14.jar, as shown in the example above.
  5. If the master index database is a SQL Server database, modify run-loader.bat as follows:
    • In the IBML Tool home directory, open the file run-loader.bat.
    • Remove %ORACLE_JDBC_JAR% and replace it with the complete path and filename of the SQL Server database driver (sqlserverdriver.jar).
    • Save and close the file.
  6. If you are using the SQL*Loader bulk loader to load the data, install SQL*Plus and SQL*Loader on the machines performing the load.
  7. If you are using the Data Integrator Bulk Loader instead of SQL*Loader to perform the load process, install NetBeans on the machines performing the load.
  8. Make sure the Java installation you are using matches the Java version used by the NetBeans installation that generated the Initial Bulk Match and Load Tool.

    Tip - To find the Java version for NetBeans, select Tools from the main menu and then select Java Platforms. To find the Java version you are using, type Java -version at a command prompt. If they do not match, either locate or install the correct version and change your PATH variable to point to the correct version.

  9. Continue to Creating the Cluster Synchronizer Database.
