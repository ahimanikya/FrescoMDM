Master Index Development Process Overview

This document provides instructions for creating a master index application using Master Index Studio. It includes creating the master index application framework, creating custom Java code, generating the application files, and creating the database. It also includes performing a data analysis using the framework created for the master index application.

The following steps outline and link to the procedures you need to follow to develop a master index application. Not all tasks are covered in this document. Where a process is covered in a separate document, the link is in italics.

  1. Perform a preliminary analysis of the data you plan to store in the master index application.

  2. Create a new Project and a new Master Index Studio application within that Project (described in Creating a Master Index Application).

  3. Define the object structure, operating environment, and certain runtime characteristics using the wizard (described in Creating a Master Index Application).

  4. Define and build custom plug-ins, and specify the plug-ins in the appropriate configuration file (described in Implementing Custom Plug-ins).

  5. Customize the configuration files (instructions for this step are provided in a separate document, Configuring Master Index Applications).

  6. Generate the master index application (described in Generating the Master Index Application).

  7. Create the database (described in Creating the Database).

    • Customize the database scripts by defining system information, processing codes, and drop-down menu values.

    • Create the database structure and any necessary indexes.

    • Define database connectivity in the application server.

  8. Perform a complete data analysis (described in Analyzing the Legacy Data).

  9. Build and deploy the Project.

  10. Define security for the EDM.
