Description

The purpose of this solution is to demonstate how a United States of America Patient model might be implemented in the Master Data Management (MDM) product suite. Specificaly the solution focuses on the  Master Index portion of MDM.

This solution uses a patient definition geared towards the United States of America. Since this is a USAPatient, fields such as the First Name, Last Name, Date of Birth and Social Security Number (SSN) are used for both blocker-search definitions and matches.

Data Model Overview.

This overview of the data model shows each Sub Object Node of the USAPatient. It highlights the main sections of the United States of America Patient model. 


Data Model

This is the Unified Modeling Language™ (UML) model used to create the USAPatient. You will notice that the model shows inheritance since the USAPatient and the Australian Patient models are very similiar. The main difference being the use of a Social Security Number (SSN) in the United States as a unique identifier and the Medicare number (MedicareNum)  in Australia as a unique identifier. Additionaly,  this model represents differences in the healthcare system reflected by the Insurance Sub Object Node of the Australian Patient.   Also, somedata is only collected in the United States (like a DriversLicence).   Finally, some inheritance relects difference in names used for an item in the United States and Australia such as the cellphone (which is called a mobilephone in Australia).

User Codes

When the user-code is selected in the MIDM GUI then the corresponding format, input_mask and value_mask are used. The input_mask shows the characters that can be typed.correspondingcorresponding

For the input_mask and value_mask:
A = an alphanumeric character [A-Za-z0-9]
D = a  decimal number [0-9]
L = an alphabetic character [A-Za-z]

For the value_mask:
^ = remove the corresponding character from the input string when storing in the data.

Example: The AUXIDDEF shows three values in the MIDM GUI, "CREDIT CARD ID", "Medical License ID" and "Household ID". If the user selects the Household ID they can type in four deciimal numbers, a space, four deciimal numbers, a space and then four deciimal numbers (example: 1111 2222 3333 4444). This will be converted to 1111222233334444 and stored in the database.
insert into sbyn_user_code (code_list, code, descr, format, input_mask, value_mask)
 values ('AUXIDDEF', 'CCID', 'CREDIT CARD ID', '[0-9]{16}', 'DDDD DDDD DDDD DDDD', 'DDDD^DDDD^DDDD^DDDD');

insert into sbyn_user_code (code_list, code, descr, format, input_mask, value_mask)
 values ('AUXIDDEF', 'MEDID', 'Medical License ID', '[A-Za-z]{3}[0-9]{6}', 'LLL-DDDDDD', 'LLL^DDDDDD');

insert into sbyn_user_code (code_list, code, descr, format, input_mask, value_mask)
 values ('AUXIDDEF', 'HHID', 'Household ID', '[A-Za-z0-9]{3}[0-9]{4}', 'AAADDDD', 'AAADDDD');
